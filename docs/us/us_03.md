# US 3 - Recommandation NutriScore
- [x] Bandeau si D ou E
- [x] Affiche nom et NutriScore recommandé
- [x] Rien si pas d'alternative
- [x] Pas de reco pour A, B, C

## Implémentation Technique
- **API** : Utilisation du endpoint de recherche `cgi/search.pl` d'OpenFoodFacts.
- **Logique** : Si le grade est D ou E, le `ProductRepository` lance une requête avec `tag_0={category}` et `tag_1={grade}` en essayant A, puis B, puis C jusqu'à trouver un résultat.
- **UI** : Condition Compose dans `ProductCard` pour afficher un bandeau `tertiaryContainer` si l'objet `betterAlternative` n'est pas nul.
